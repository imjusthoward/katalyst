# Architecture

## Product role

Katalyst is meant to improve how distributed organizations:

- access information
- preserve continuity
- coordinate activity
- execute reliably under real constraints

## Initial deployment context

Tutoring network in Japan:

- students
- tutors
- operations coordinators
- sessions
- announcements
- resources
- notes and continuity context

## Core layers

### Access layer

- searchable resources
- announcements
- dashboards
- role-based views

### Continuity layer

- handover notes
- recurring templates
- cycle archives
- decision logs

### Execution layer

- projects
- tasks
- owners
- due dates
- recurring workflows

### Network layer

- people
- teams
- organizations
- memberships
- roles

## Technical direction

- Next.js
- TypeScript
- Tailwind
- Supabase
- Vercel
- Sentry
- PostHog

## Non-goals for the first milestone

- organization-wide AI agents everywhere
- enterprise-grade permission matrix complexity
- public social features
- shiny branding work that delays the operational core
